Lesson 1: The Basics of the Web and HTML

HTML Basics

HTML stands for Hyper Text Markup Language. which is what defines most web pages.

HTML is made up of the following...
Text content shows what you see.
Markup shows what your page will look like.
References to other documents, such as images are possible.
Links to other web pages are possible too.

Tags

HTML uses tags in a certain structure to create a webpage, such as !doctype, head, title, body, etc.
Tags are what HTML uses to mark text in a certain way.
A tag normally has an opening tag, text, then a closing tag.
Tags that do not contain content are called "void" tags.
I learned how to use bold and to provide emphasis via the italics tags.

Inline vs. Block

Tags can be either inline or block.
Inline tags examples are break, bold, emphasis, void tags, new lines, etc.
Block tags create an invisible block around the content. Examples of this is paragraph and div.

Attributes

Attributes, ATTR equal a value.
The anchor tag creates links to other websites.
I can add images using the image tag and src attribute.

Lesson 3: Adding CSS Style to HTML Structure

CSS Cascading Style Sheets

CSS- Cascading Style Sheets avoid program repitition. A CSS file can be used via the link tag. Or the same thing can be accomoplished via the style HTMl tag.

Inheritance is the use of DOM to relay properties between ancestors and descendents. Descendents can get text based properties, but not box-related properties. Inheritance cuts down on repitition, uses less CSS, faster web page loading, saves money on bandwidth and developmental costs.

Text based examples are color, font, line spacing, line height, list style, text align, text indent, visibility, etc.

Box based examples are background, border, display, float and clear, height and width, margin, outline, overflow, padding, position, etc.

Selector decides what style gets applied. It is defined by a class name, along with attributes and values.

Box Model consists of a margin, border, padding, and content. Padding can be adjusted by background. Border can be adjusted by the color property. Margin is the space between boxes. Width consists of content + left and right border + left and right margin.

Vaiables

Variable is a name that is a replacement for a value.
Assign variables by using the equal sign. In the world of variables, equal does not mean the same as a mathmatical equal sign. It simply means it has been assigned a value.
Values can be literal via the use of quotation marks. For example, value = "4" + "4", would produce an outcome of 44. Without the quotation marks, the values would be added together mathmatically. So, value = 4 + 4, would produce an outcome of 8.

Strings

Strings are like variables, but surrounded by quotation marks. It does not matterif single or double quotation marks are used, as long as they match. If no quotation marks are used, the computer thinks the entity is a variable.
Operators mean something different with strings, than their original mathmatical purpose. Plus means to concantenate values. Multiplication sign defines a quantity.
Selecting Sub Sequences is a way to ask the computer to define a location within a string.
Square Brackets are used to request this location. The computer numbers each value starting at 0 for this position identifier. A value can be requested on it own. It can be calculated. It can be defined as a range by using a colon within the square bracket. When a range is defined, it does NOT include the last value. If you leave the range request open ended, then the last value is included. If no value meets the request criteria, then a negative one is the result.
Indexes can be used in addition to the position request. Position requests will find the first item match. An index can be defined to tell the computer to find the first match, but only after the postion number listed as an index. This method can be used to find and replace things within a string.

Input, Functions, Output

Functions are programming statements/routines that can use multiple inputs to produce an output. The concept of multiple inputs makes functions attractive to programmers by performing repeated actions without rewriting code.
Defining a function is done by using the def command, a function name, and a parameter, ending in a colon. The quantity of inputs is defined via the parameter portion of the function. Functions allow for multiple inputs. This allows programmers to set parameter values based on outputs of other functions.
Using a function is setting the values of the input. It is returning the output or result. Functions are valuable because it allows processes or procedures to be performed using a small amount of code. Functions can be linked to perform multiple actions.
Return command tells the function to produce output. Without it, the fuunction still performs it's action. It just does not show the results. If the result does not exist or there is no answer, the function will have NONE for its output.
Procedure Composition is the term used to describe the activity of using procedures within procedures.

Comparison Operators such as greater than, less than, equal to are used to compare variables and values. An equality comparison uses two == signs. Mutation and Aliasing

Mutation is the concept of changing the value of a list after its creation.
This is different than data manipulation where a string value can be redefined, but not really mutated.
Lists items are defined as though they are in individual boxes, which are subject to mutation.
As mutation happens it is important to note changes happen to the data. But the list is still the same like, but with a different value associated.
Aliasing is when multiple variable values point to the same list. As the list is changed, both variables are actually affected.
This is important to understand for when values do not come out as expected. Aliasing could be the cause and must be acknowledged.

Object Oriented Programming

Definition from Wikipedia Object-oriented programming (OOP) is a programming paradigm based on the concept of "objects", which are data structures that contain data, in the form of fields, often known as attributes; and code, in the form of procedures, often known as methods. A distinguishing feature of objects is that an object's procedures can access and often modify the data fields of the object with which they are associated (objects have a notion of "this"). In OO programming, computer programs are designed by making them out of objects that interact with one another.[1][2] There is significant diversity in object-oriented programming, but most popular languages are class-based, meaning that objects are instances of classes, which typically also determines their type.
Definition from Dictionary.com A schematic paradigm for computer programming in which the linear concepts of procedures and tasks are replaced by the concepts of objects and messages. An object includes a package of data and a description of the operations that can be performed on that data. A message specifies one of the operations, but unlike a procedure, does not describe how the operation should be carried out.
Definition from PCmag.com Writing software that supports a model wherein the data and their associated processing (called "methods") are defined as self-contained entities called "objects." Object-oriented programming (OOP) languages, such as C++ and Java, provide a formal set of rules for creating and managing objects. The data are stored in a traditional relational database or in an object database if the data have a complex structure. There are three major features in object-oriented programming: encapsulation, inheritance and polymorphism.
